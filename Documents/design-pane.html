Project Display Pane

When "New Project" is selected from either the main menu or the tool bar, the design pane replaces the main window. The needed command buttons and menu items activate from their grayed-out state. The design pane consists of two parts. There is the user data entry area. Next to it, is a text viewer that shows a brief description of the design options. The divider between the data entry pane and the text viewer is resizable by dragging the vertical divider.

 

The design choices are shown via a drop-down menu that appears when the button is clicked. Items may be selected more than once for convenient review of the descriptions.

The text fields accept up to 16 characters. Values may be made in terms of Hz, KHz, or MHz for frequency, and K or MEG for impedances. These values are appropriately scaled for performing the calculations. No trailing modifiers default to Hz and ohms. Values may also be entered in exponential notation, for example: 7.0E6 instead of 7.0MHz. These text fields may be cycled through in order via the <TAB> key.

The command buttons also activate at this time. The "OK" button finalizes the project. If entries are required, but not supplied, an error advisory dialog will alert the user as to the situation. The missing data will receive focus, so that the correct value(s) may be entered. The "Clear" button removes text from all the text fields.

 

Once the project is complete, the welcome screen image is replaced by the data image. This includes a schematic of the selected filter, and the component values. This is a GIF image, and may be saved as such. It is viewable from any image viewer that can display GIFs (almost all of them). The image may also be printed out as hard copy. There is a potential pitfall to doing so: some printers these days aren't Post Script printers: the format FOX uses to print files. As an alternative, the design data may also be printed as text only. There is a possibility of saving to a Post Script file that can be converted by other means.

After project completion, a new project can be started without having to exit and restart the application. All previous data will be lost, and if not saved, the user will be prompted to save the current project before starting a new one if it hasn't been saved already.
